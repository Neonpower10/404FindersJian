# Functionele requirements – User stories

## FR-010: Maatwerkmodules – Ondersteuning van maatwerkmodules per camping
•	Als campingbeheerder wil ik extra modules kunnen toevoegen of aanzetten, zodat ik mijn camping specifieke functies kan geven.
DoR
•	Er is duidelijk welke extra modules beschikbaar moeten zijn.
DoD
•	Beheerder kan modules activeren en deactiveren via de interface.

## FR-019: Meerdere talen – Minimaal Nederlands en Engels
•	Als gast wil ik de interface in mijn eigen taal kunnen gebruiken, zodat ik alles goed begrijp.
•	Als campingbeheerder wil ik content in meerdere talen beheren, zodat ik ook internationale gasten kan helpen.
DoR
•	De te ondersteunen talen zijn gedefinieerd (minimaal NL en EN).
DoD
•	Gast kan in de interface schakelen tussen NL en EN.
 
## FR-020: Mobiel – Goed werken op mobiel en tablet
•	Als gast wil ik de applicatie makkelijk kunnen gebruiken op mijn smartphone of tablet, zodat ik niet afhankelijk ben van een computer.
•	Als campingbeheerder wil ik het systeem ook mobiel beheren, zodat ik taken overal kan uitvoeren.
DoR
•	Mockups of wireframes voor mobiele weergave zijn beschikbaar.
DoD
•	Alle functionaliteiten zijn bruikbaar op smartphone en tablet.
•	UI schaalt correct mee en knoppen/velden zijn goed bedienbaar met touch.
Niet-functionele requirements – User stories

## NFR-008: Responsief – Automatisch aanpassen aan scherm (telefoon, tablet, laptop)
•	Als gast wil ik dat de interface zich automatisch aanpast aan mijn apparaat, zodat ik altijd een goede gebruikerservaring heb.
DoR
•	Schermformaten en breakpoints zijn vastgelegd.
•	Ontwerp richtlijnen voor responsiviteit zijn beschikbaar.
DoD
•	Layout schaalt automatisch correct op telefoon, tablet en laptop.
