# Standaarden
## GitHub
* documentatie (zoals DailyStandUp, Sprint review, Retrospective) in Nederlands
* Iedere dag dat we op locatie zijn doen we een stand up (wordt elke keer door iemand anders geleid) 
* Bestanden worden onder in de juiste map(pen) gezet

## Git Workflow
* Voor elke issue een nieuwe branch
* De branch geef je de naam van de userstory nummer en feature die je uitwerkt, of in het geval van een bug fix vermeld je de bug
* Commit je werk met regelmaat, zelfs als het nog niet werkt
* Bij een pull request voegen we altijd duidelijk toe wat er in de pull request wordt toegevoegd, weggehaald of veranderd
* Vergeet niet om ook daadwerkelijk review te vragen aan de anderen (kopje Review rechtsboven in en klik dan de teamgenoten)

## Development
* Variabelen (en bestand) namen in CamelCase en in het Engels
* Waar mockdata wordt toegepast wordt genoteerd
* Voeg docstrings toe, deze zijn in het Engels
* Voorkom spaghetti code door niet al te grote functies/klassen te gebruiken, maar ze op te splitsen in kleinere functies/klassen die je aanroept
* Werk op een logische volgorde die consistent in het hele project blijft
